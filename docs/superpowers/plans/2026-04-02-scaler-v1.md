# Scaler V1 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Rust CLI that runs host commands under constrained resources on Ubuntu and macOS with a unified command shape, backend capability reporting, and a live monitor.

**Architecture:** Keep the binary thin and put the contract in a library crate. The library owns CLI parsing, normalized resource values, backend selection, capability reporting, run-loop coordination, output framing, and UI rendering. Ubuntu starts with a `systemd-run --scope` backend, macOS starts with a `taskpolicy` backend, and both feed the same `core` lifecycle and monitoring path.

**Tech Stack:** Rust stable, `clap`, `anyhow`, `thiserror`, `rust_decimal`, `ratatui`, `crossterm`, `assert_cmd`, `predicates`, `tempfile`

impl CpuLimit {
    pub fn parse(raw: &str) -> anyhow::Result<Self> {
    // 1. strip and lowercase
    // 2. require trailing 'c'
    // 3. parse the numeric prefix with rust_decimal
    // 4. multiply by 100 and half-up round to centi-cores
    // 5. reject values below 1 centi-core or above u32::MAX centi-cores
    }
}

impl MemoryLimit {
    pub fn parse(raw: &str) -> anyhow::Result<Self> {
    // 1. strip and lowercase
    // 2. require a single-unit suffix in b|k|m|g|t
    // 3. parse the numeric prefix with rust_decimal
    // 4. multiply by the 1024-based unit multiplier and half-up round to bytes
    // 5. reject values below 1 MiB or above u64::MAX bytes
    }
}

In `src/cli/args.rs`, model the command contract with `clap`:

```rust
#[derive(clap::Parser, Debug)]
pub struct Cli {
    #[command(subcommand)]
    pub command: Command,
}

#[derive(clap::ValueEnum, Debug, Clone, Copy)]
pub enum InteractiveModeArg { Auto, Always, Never }

#[derive(clap::ValueEnum, Debug, Clone, Copy)]
pub enum ShellArg { Sh, Bash, Zsh }

#[derive(clap::Subcommand, Debug)]
pub enum Command {
    Run(RunCommand),
    Doctor,
    Version,
}

#[derive(clap::Args, Debug)]
pub struct RunCommand {
    #[arg(long, value_parser = crate::cli::values::parse_cpu_limit)]
    pub cpu: Option<CpuLimit>,
    #[arg(long, value_parser = crate::cli::values::parse_memory_limit)]
    pub mem: Option<MemoryLimit>,
    #[arg(long, value_enum, default_value_t = InteractiveModeArg::Auto)]
    pub interactive: InteractiveModeArg,
    #[arg(long)]
    pub shell: Option<ShellArg>,
    #[arg(long = "no-monitor", default_value_t = true, action = clap::ArgAction::SetFalse)]
    pub monitor: bool,
    #[arg(last = true)]
    pub trailing: Vec<String>,
}
```

In `src/cli/mod.rs`, add an argv preprocessor:

```rust
pub fn normalize_argv(raw: Vec<std::ffi::OsString>) -> Vec<std::ffi::OsString> {
    // raw[0] is the binary name
    // if raw.len() == 1, return raw unchanged
    // if raw[1] is exactly "run", "doctor", or "version", return raw unchanged
    // otherwise insert "run" at index 1 and return the rewritten argv
}

pub fn parse_from(raw: Vec<std::ffi::OsString>) -> anyhow::Result<Cli> {
    let normalized = normalize_argv(raw);
    Ok(Cli::try_parse_from(normalized)?)
}
```

Use the preprocessor so:

- `scaler --cpu 1c -- echo ok` becomes `scaler run --cpu 1c -- echo ok`
- `scaler doctor` stays unchanged
- bare `scaler` still fails as usage error
- `scaler run --` fails because `RunCommand.trailing` is empty after the required delimiter
- `scaler run --shell sh -- echo ok` fails because shell mode must preserve exactly one script token
- all integration tests that exercise shorthand must call `scaler::cli::parse_from(...)`, not `Cli::try_parse_from(...)` directly

Model `RunCommand` with a validator that enforces:

- if `shell` is `None`, `trailing.len() >= 1`

- `trailing` is only accepted after the explicit `--` delimiter

    next

}

impl Default for OutputCollector {

        Self { next_sequence: 0 }
    }
}

        OutputFrame {
            sequence: next_sequence(&mut self.next_sequence),
            captured_at: std::time::SystemTime::now(),
            stream: OutputStream::Stdout,
            bytes,
        }

            stream: OutputStream::Stderr,

            sequence: next_sequence(&mut self.next_sequence),
